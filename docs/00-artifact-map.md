Artifact Map (Start Here)
Use this map to navigate the toolkit in the recommended order: tiering → evidence → baselines → patterns → exceptions → monitoring → metrics.

Start-here flow
1) Tier the supplier
Tiering decision tree: tiering-model/supplier-tiering-decision-tree.md
Tiering intake: tiering-model/Supplier Tiering Intake.pdf
Tiering folder readme: tiering-model/readme.md
Output: Tier = Critical / Important / Low + documented rationale.

2) Issue the evidence pack (due diligence)
Tier 1 evidence pack: tiering-model/Tier 1 Supplier Evidence Pack.pdf
Tier 2 evidence pack: evidence-packs/Tier 2 Important Supplier Evidence Pack .docx
Tier 3 lightweight checks: evidence-packs/Tier 3 Low Supplier Checks — Lightweight Assurance.docx
Supplier response template: evidence-packs/Supplier Evidence Pack Response Template.docx
Tier 1 internal assurance checklist: evidence-packs/Tier 1 Critical Supplier Checks — Assurance Checklist.docx
Output: Evidence received + gaps identified.

3) Apply control baselines (what to require + how to validate)
IAM baseline: control-baselines/IAM baseline.pdf
Logging & auditability baseline: control-baselines/Logging and Auditability Control.pdf
API baseline: control-baselines/API Baseline.pdf
Connectivity baseline: control-baselines/Connectivity Baseline.pdf
Vulnerability management baseline: control-baselines/Vuln Mgmt baseline.pdf
Incident notification baseline: control-baselines/Incident Notification Baseline.pdf
Resilience & recovery baseline: control-baselines/Resilience and Recovery Baseline.pdf
Output: Pass/partial/gap with evidence links and remediation actions.

4) Select integration patterns (how to onboard safely)
SSO pattern: integration-patterns/SSO.pdf
RBAC template: integration-patterns/RBAC.pdf
API integration pattern: integration-patterns/API Integration.pdf
Logging & audit export pattern: integration-patterns/Logging and Audit Export.pdf
VPN/private link pattern: integration-patterns/VPN Connectivity.pdf
Output: Approved pattern(s) + onboarding validation evidence.

5) Governance + exceptions (make decisions explicit)
Program charter: program-charter/Program Charter.pdf
Governance/approvals: program-charter/Governance Approvals.pdf
Minimum security by tier: program-charter/Minimum Security for Vendor Tiers.pdf
Policy exceptions standard: program-charter/Policy Exceptions.pdf
Exception request template: program-charter/Policy Exception request.docx
RACI: program-charter/RACI.pdf
Data retention: program-charter/Data Retention and Disposal Policy.pdf
Output: Decision record + exceptions (if any) with expiry and compensating controls.

6) Contract it (make it enforceable)
Contract overview by tier: contracts/Contract Overview by Vendor Tier.pdf
Security schedule template: contracts/Security Schedule Template.docx
Security schedule (Critical-tier): contracts/Security Schedule — Critical Tier Tier 1 Vendor Template.docx
DPA checklist: contracts/DPA Checklist.docx
Audit & assurance: contracts/Audit Clauses.pdf
Sub-processor clauses: contracts/Sub-Processor Clauses.pdf
Incident notification clause: contracts/Incident Notification Clause.pdf
Resilience/DR clause: contracts/DR Clause.pdf
Offboarding clauses: contracts/OffBoarding Clauses.pdf
Output: Executed contract aligned to the supplier’s tier and scope.

7) Monitor, renew, and respond to incidents
Review cadence: monitoring/Review Cadence.pdf
Trigger events (out-of-cycle reassessment): monitoring/Trigger Events.pdf
Ongoing assurance checklist: monitoring/Ongoing Assurance Checklist.docx
Renewal assessment playbook: monitoring/Renewal Assessment Playbook.docx
Incident response (supplier-facing):

Supplier incident management playbook: monitoring/Supplier Incident Management Playbook.pdf
Incident-triggered supplier review playbook: monitoring/Incident Triggered Supplier Review Playbook.pdf
First 60-minute questions: monitoring/Supplier Incident First 60-Minute Questions.docx
Full incident information request: monitoring/Supplier Incident Information Request.docx
Output: Updated supplier posture, actions tracked to closure, and reassessment evidence retained.

8) Report it (KPIs/KRIs and packs)
Reporting pack guidance: metrics/Reporting Pack Guidance.pdf
Weekly operational scoreboard template: metrics/Weekly Operational Scoreboard — Template.docx
(Program-level KPI/KRI references currently sit in program-charter/):

KPI/KRI set: program-charter/KPI_KRI.pdf
Monthly dashboard template: program-charter/Monthly Third-Party Security Dashboard.docx
Output: Weekly throughput control + monthly risk posture reporting.
